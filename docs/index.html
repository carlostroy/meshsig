MeshSig

Cryptographic security layer for AI agents. Every agent gets an identity. Every message is signed. Every delegation is verified. Transparent proxy — zero changes to your agents.

Why MeshSig?

🔐

Ed25519 Signatures

Every agent-to-agent message is cryptographically signed. No one can impersonate an agent. No one can tamper with a message.

🪪

W3C DID Identity

Each agent gets a decentralized identifier (did:msig:...) following the W3C standard. Portable, self-sovereign, framework-agnostic.

🔀

Transparent Proxy

Intercepts all agent traffic via iptables — zero changes to your agents or gateway. One command to deploy. Works with any framework.

🏢

3D Office Dashboard

Isometric office with LEGO-style agents that walk, glow, and exchange messages in real-time. Pan, zoom, and watch your agents work.

🤝

Verified Handshakes

Mutual challenge-response authentication between agents. Both sides prove identity before any trust is established.

📊

Audit Trail

Every signed message logged with tamper-proof cryptographic proof. Export JSON for compliance. Trust scores updated in real-time.

How It Works

WITHOUT MESHSIG:
  Agent → localhost:3001 → Gateway → executes

WITH MESHSIG:
  Agent → localhost:3001 → [iptables] → MeshSig:4888 [SIGN ✓] → Gateway:3001 → executes
                                                ↓
                                       Dashboard + Audit

Deploy in one command:
  bash scripts/deploy-proxy.sh 3001
